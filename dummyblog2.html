Popular Commands in Git

Author:RamaChandran

what is Git?

It is a distributed version control system. It is mainly used to track the changes in the project. Git is created by Linux Torvalds the man behind Linux OS. The main focus of git is to track when the changes are made, by whom the changes are made, and for what reason the changes are made.

Some of the git providers are Github, BitBucket, Gitlab, etc. The only difference between the git and git providers are git store the changes in the cloud whereas git providers store it in cloud.Some of the commonly used git commands are

1) git init

This will initialize the git for the project. It will create a file called .git which contains all the history of the project. The "." before the file name represents that it is a hidden file.

Syntax

git init

2) git add

This command is used to add the changes to the staging area before committing. The staging area is used to review the changes made by us. The option specify whether you want to stage all the changes or particular file

Syntax

git add options

3) git status

This command gives the status such as what are the changes that are needed to be staged and what are the changes that are needed to be committed.

Syntax

git status

4) git commit

This command is used to commit the changes which are staged. Giving the appropriate git message is also a good practice like

  • fix: Fixing the bug
  • feat: adding a new feature
  • test: All the things related to testing
  • docs: All the things related to documentation
  • style: All the things related to styling like color, font family etc

Syntax

git commit options message

5) git push

This is used to push all the changes to GitHub which are committed

Syntax

git push
